class  RobotContainer
 This class is where the bulk of the robot should be declared. Since Command-based is a "declarative" paradigm, very little robot logic should actually be handled in the Robot periodic methods (other than the scheduler calls). Instead, the structure of the robot (including subsystems, commands, and button mappings) should be declared here. More...
 

Detailed Description

This file declares the RobotContainer class.

===========================================================================

The RobotContainer class holds robot subsystems and commands, and is where most of the declarative robot setup (e.g. button bindings) is performed.
