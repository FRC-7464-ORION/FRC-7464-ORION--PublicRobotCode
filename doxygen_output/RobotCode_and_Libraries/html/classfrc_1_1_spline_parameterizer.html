◆ kMaxIterations

constexpr int frc::SplineParameterizer::kMaxIterations = 5000
staticprivate

A malformed spline does not actually explode the LIFO stack size. Instead, the stack size stays at a relatively small number (e.g. 30) and never decreases. Because of this, we must count iterations. Even long, complex paths don't usually go over 300 iterations, so hitting this maximum should definitely indicate something has gone wrong.
