233 // The joystick/motor curve parameters should make it easier to control
234 // motors with a joystick and/or thumbstick and/or other variable
235 // control (such as the triggers on some controllers).
236 //
237 // Here is what each constant/parameter means/does:
238 //
239 // Null Zone: Sometimes called deadzone. Any absolute value output from the
240 // joystick/thumbstick/trigger/etc. that is less than the
241 // null zone value will translate into zero output onto a motor.
242 // This keeps the motor from "jittering", especially if the
243 // output of the controller is noisy (or a nervous driver ;-).
244 //
245 // Typical values for a null zone are from 0.0 to 0.2.
246 // The null zone cannot exceed 1.0 (nothing will work!).
247 //
248 // Constant name: k_DriveTrainAbsNullZone
249 //
250 // Max Speed: This is the maximum speed that the motor will output
251 // (turbo off) (either forward or reverse) in non-turbo mode.
252 // Any absolute value output from the
253 // joystick/thumbstick/trigger/etc. that is greater than the
254 // maximum speed will be limited to the maximum speed
255 // (either forward or reverse) defined by this constant
256 // parameter.
257 //
258 // Set the maximum speed to the percentage of full speed desired.
259 // For example, to limit the motor to half speed, set max speed
260 // to 50% (0.50). Similar to the null zone, don't set the
261 // max speed to 0.0, or again nothing will work!
262 //
263 // Constant name: k_DriveTrainAbsValMaxSpeedNoTurbo
264 //
265 // Max Speed: This is the maximum speed that the motor will output
266 // (turbo on) (either forward or reverse) in turbo mode.
267 // Any absolute value output from the
268 // joystick/thumbstick/trigger/etc. will be multiplied
269 // by the ratio of max_speed_turbo_on to
270 // max_speed_turbo_off. By using this ratio, the maximum
271 // speed will be limited to the maximum turbo speed, but
272 // providing boosted speed at all other speeds as well.
273 //
274 // Constant name: k_DriveTrainAbsValMaxSpeedTurboOn
275 //
276 // Exponent: This is the exponent of a curve used on the joystick input
277 // to motor output mapping. Its intent is to make the motor more
278 // responsive at lower speeds, at the loss of fine control
279 // at higher speeds. An exponent of 1.0 makes the output
280 // linear (like a straight line). Higher exponents provide
281 // for better control at lower speeds, but sacrifice control
282 // at higher speeds.
283 //
284 // You'll typically want the exponent to be between 2.0-4.0.
285 // Anything past 5.0 will result in an uncontrollable robot!
286 // Do not use an exponent less than 1.0. The math breaks
287 // down, and we may open a rift to another universe.
288 //
289 // Constant name: k_DriveTrainExponent
290 //
291 // d: This is the delay value of an infinite impulse response (IIR)
292 // digital low pass filter (DLPF). Its intent is to make the motor
293 // output less "jerky" (i.e. more smooth) over time, at the
294 // expense of introducing some delay from the joystick input
295 // to the motor output.
296 //
297 // The value d must be constrained to 0.00 <= d <= 1.00.
298 //
299 // As the value of d decreases, the more smoothing (but more
300 // delay) is introduced. As d increases, less delay and
301 // less smoothing is used.
302 //
303 // NOTE: Motor controllers for the drive train should be
304 // put in "BRAKE" mode, and not in "COAST" mode for
305 // this to work properly.
306 //
307 // For more information about IIR DLP filters, and how they
308 // belong to a branch of engineering mathematics called
309 // Digital Signal Processing (DSP), see the following:
310 //
311 // https://en.wikipedia.org/wiki/Digital_signal_processing
312 // https://en.wikipedia.org/wiki/Infinite_impulse_response
313 // https://www.dsprelated.com/freebooks/filters/Introduction.html
314 // https://www.embedded.com/design/configurable-systems/4025591/Digital-filtering-without-the-pain
315 //
316 // Constant name: k_DriveTrain_IIR_DLPF_d
317 //
318 // See the SW_Design repository, and look at NewThumbsticksOutputs.xlsx
319 // and IIR_DLPF.xlsx for some graphs that illustrate these concepts.
320
