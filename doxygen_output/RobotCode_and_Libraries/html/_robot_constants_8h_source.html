223 // The joystick/motor curve parameters should make it easier to control
224 // motors with a joystick and/or thumbstick and/or other variable
225 // control (such as the triggers on some controllers).
226 //
227 // Here is what each constant/parameter means/does:
228 //
229 // Null Zone: Sometimes called deadzone. Any absolute value output from the
230 // joystick/thumbstick/trigger/etc. that is less than the
231 // null zone value will translate into zero output onto a motor.
232 // This keeps the motor from "jittering", especially if the
233 // output of the controller is noisy (or a nervous driver ;-).
234 //
235 // Typical values for a null zone are from 0.0 to 0.2.
236 // The null zone cannot exceed 1.0 (nothing will work!).
237 //
238 // Constant name: k_DriveTrainAbsNullZone
239 //
240 // Max Speed: This is the maximum speed that the motor will output
241 // (turbo off) (either forward or reverse) in non-turbo mode.
242 // Any absolute value output from the
243 // joystick/thumbstick/trigger/etc. that is greater than the
244 // maximum speed will be limited to the maximum speed
245 // (either forward or reverse) defined by this constant
246 // parameter.
247 //
248 // Set the maximum speed to the percentage of full speed desired.
249 // For example, to limit the motor to half speed, set max speed
250 // to 50% (0.50). Similar to the null zone, don't set the
251 // max speed to 0.0, or again nothing will work!
252 //
253 // Constant name: k_DriveTrainAbsValMaxSpeedNoTurbo
254 //
255 // Max Speed: This is the maximum speed that the motor will output
256 // (turbo on) (either forward or reverse) in turbo mode.
257 // Any absolute value output from the
258 // joystick/thumbstick/trigger/etc. will be multiplied
259 // by the ratio of max_speed_turbo_on to
260 // max_speed_turbo_off. By using this ratio, the maximum
261 // speed will be limited to the maximum turbo speed, but
262 // providing boosted speed at all other speeds as well.
263 //
264 // Constant name: k_DriveTrainAbsValMaxSpeedTurboOn
265 //
266 // Exponent: This is the exponent of a curve used on the joystick input
267 // to motor output mapping. Its intent is to make the motor more
268 // responsive at lower speeds, at the loss of fine control
269 // at higher speeds. An exponent of 1.0 makes the output
270 // linear (like a straight line). Higher exponents provide
271 // for better control at lower speeds, but sacrifice control
272 // at higher speeds.
273 //
274 // You'll typically want the exponent to be between 2.0-4.0.
275 // Anything past 5.0 will result in an uncontrollable robot!
276 // Do not use an exponent less than 1.0. The math breaks
277 // down, and we may open a rift to another universe.
278 //
279 // Constant name: k_DriveTrainExponent
280 //
281 // d: This is the delay value of an infinite impulse response (IIR)
282 // digital low pass filter (DLPF). Its intent is to make the motor
283 // output less "jerky" (i.e. more smooth) over time, at the
284 // expense of introducing some delay from the joystick input
285 // to the motor output.
286 //
287 // The value d must be constrained to 0.00 <= d <= 1.00.
288 //
289 // As the value of d decreases, the more smoothing (but more
290 // delay) is introduced. As d increases, less delay and
291 // less smoothing is used.
292 //
293 // NOTE: Motor controllers for the drive train should be
294 // put in "BRAKE" mode, and not in "COAST" mode for
295 // this to work properly.
296 //
297 // For more information about IIR DLP filters, and how they
298 // belong to a branch of engineering mathematics called
299 // Digital Signal Processing (DSP), see the following:
300 //
301 // https://en.wikipedia.org/wiki/Digital_signal_processing
302 // https://en.wikipedia.org/wiki/Infinite_impulse_response
303 // https://www.dsprelated.com/freebooks/filters/Introduction.html
304 // https://www.embedded.com/design/configurable-systems/4025591/Digital-filtering-without-the-pain
305 //
306 // Constant name: k_DriveTrain_IIR_DLPF_d
307 //
308 // See the SW_Design repository, and look at NewThumbsticksOutputs.xlsx
309 // and IIR_DLPF.xlsx for some graphs that illustrate these concepts.
