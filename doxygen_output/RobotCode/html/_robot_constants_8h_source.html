193 // The joystick/motor curve parameters should make it easier to control
194 // motors with a joystick and/or thumbstick and/or other variable
195 // control (such as the triggers on some controllers).
196 //
197 // Here is what each constant/parameter means/does:
198 //
199 // Null Zone: Sometimes called deadzone. Any absolute value output from the
200 // joystick/thumbstick/trigger/etc. that is less than the
201 // null zone value will translate into zero output onto a motor.
202 // This keeps the motor from "jittering", especially if the
203 // output of the controller is noisy (or a nervous driver ;-).
204 //
205 // Typical values for a null zone are from 0.0 to 0.2.
206 // The null zone cannot exceed 1.0 (nothing will work!).
207 //
208 // Constant name: k_DriveTrainAbsNullZone
209 //
210 // Max Speed: This is the maximum speed that the motor will output
211 // (turbo off) (either forward or reverse) in non-turbo mode.
212 // Any absolute value output from the
213 // joystick/thumbstick/trigger/etc. that is greater than the
214 // maximum speed will be limited to the maximum speed
215 // (either forward or reverse) defined by this constant
216 // parameter.
217 //
218 // Set the maximum speed to the percentage of full speed desired.
219 // For example, to limit the motor to half speed, set max speed
220 // to 50% (0.50). Similar to the null zone, don't set the
221 // max speed to 0.0, or again nothing will work!
222 //
223 // Constant name: k_DriveTrainAbsValMaxSpeedNoTurbo
224 //
225 // Max Speed: This is the maximum speed that the motor will output
226 // (turbo on) (either forward or reverse) in turbo mode.
227 // Any absolute value output from the
228 // joystick/thumbstick/trigger/etc. will be multiplied
229 // by the ratio of max_speed_turbo_on to
230 // max_speed_turbo_off. By using this ratio, the maximum
231 // speed will be limited to the maximum turbo speed, but
232 // providing boosted speed at all other speeds as well.
233 //
234 // Constant name: k_DriveTrainAbsValMaxSpeedTurboOn
235 //
236 // Exponent: This is the exponent of a curve used on the joystick input
237 // to motor output mapping. Its intent is to make the motor more
238 // responsive at lower speeds, at the loss of fine control
239 // at higher speeds. An exponent of 1.0 makes the output
240 // linear (like a straight line). Higher exponents provide
241 // for better control at lower speeds, but sacrifice control
242 // at higher speeds.
243 //
244 // You'll typically want the exponent to be between 2.0-4.0.
245 // Anything past 5.0 will result in an uncontrollable robot!
246 // Do not use an exponent less than 1.0. The math breaks
247 // down, and we may open a rift to another universe.
248 //
249 // Constant name: k_DriveTrainExponent
250 //
251 // d: This is the delay value of an infinite impulse response (IIR)
252 // digital low pass filter (DLPF). Its intent is to make the motor
253 // output less "jerky" (i.e. more smooth) over time, at the
254 // expense of introducing some delay from the joystick input
255 // to the motor output.
256 //
257 // The value d must be constrained to 0.00 <= d <= 1.00.
258 //
259 // As the value of d decreases, the more smoothing (but more
260 // delay) is introduced. As d increases, less delay and
261 // less smoothing is used.
262 //
263 // NOTE: Motor controllers for the drive train should be
264 // put in "BRAKE" mode, and not in "COAST" mode for
265 // this to work properly.
266 //
267 // For more information about IIR DLP filters, and how they
268 // belong to a branch of engineering mathematics called
269 // Digital Signal Processing (DSP), see the following:
270 //
271 // https://en.wikipedia.org/wiki/Digital_signal_processing
272 // https://en.wikipedia.org/wiki/Infinite_impulse_response
273 // https://www.dsprelated.com/freebooks/filters/Introduction.html
274 // https://www.embedded.com/design/configurable-systems/4025591/Digital-filtering-without-the-pain
275 //
276 // Constant name: k_DriveTrain_IIR_DLPF_d
277 //
278 // See the SW_Design repository, and look at NewThumbsticksOutputs.xlsx
279 // and IIR_DLPF.xlsx for some graphs that illustrate these concepts.
