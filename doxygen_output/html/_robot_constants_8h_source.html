121 // The joystick/motor curve parameters should make it easier to control
122 // motors with a joystick and/or thumbstick and/or other variable
123 // control (such as the triggers on some controllers).
124 //
125 // Here is what each constant/parameter means/does:
126 //
127 // Null Zone: Sometimes called deadzone. Any absolute value output from the
128 // joystick/thumbstick/trigger/etc. that is less than the
129 // null zone value will translate into zero output onto a motor.
130 // This keeps the motor from "jittering", especially if the
131 // output of the controller is noisy (or a nervous driver ;-).
132 //
133 // Typical values for a null zone are from 0.0 to 0.2.
134 // The null zone cannot exceed 1.0 (nothing will work!).
135 //
136 // Constant name: k_DriveTrainAbsNullZone
137 //
138 // Max Speed: This is the maximum speed that the motor will output
139 // (turbo off) (either forward or reverse) in non-turbo mode.
140 // Any absolute value output from the
141 // joystick/thumbstick/trigger/etc. that is greater than the
142 // maximum speed will be limited to the maximum speed
143 // (either forward or reverse) defined by this constant
144 // parameter.
145 //
146 // Set the maximum speed to the percentage of full speed desired.
147 // For example, to limit the motor to half speed, set max speed
148 // to 50% (0.50). Similar to the null zone, don't set the
149 // max speed to 0.0, or again nothing will work!
150 //
151 // Constant name: k_DriveTrainAbsValMaxSpeedNoTurbo
152 //
153 // Max Speed: This is the maximum speed that the motor will output
154 // (turbo on) (either forward or reverse) in turbo mode.
155 // Any absolute value output from the
156 // joystick/thumbstick/trigger/etc. will be multiplied
157 // by the ratio of max_speed_turbo_on to
158 // max_speed_turbo_off. By using this ratio, the maximum
159 // speed will be limited to the maximum turbo speed, but
160 // providing boosted speed at all other speeds as well.
161 //
162 // Constant name: k_DriveTrainAbsValMaxSpeedTurboOn
163 //
164 // Exponent: This is the exponent of a curve used on the joystick input
165 // to motor output mapping. Its intent is to make the motor more
166 // responsive at lower speeds, at the loss of fine control
167 // at higher speeds. An exponent of 1.0 makes the output
168 // linear (like a straight line). Higher exponents provide
169 // for better control at lower speeds, but sacrifice control
170 // at higher speeds.
171 //
172 // You'll typically want the exponent to be between 2.0-4.0.
173 // Anything past 5.0 will result in an uncontrollable robot!
174 // Do not use an exponent less than 1.0. The math breaks
175 // down, and we may open a rift to another universe.
176 //
177 // Constant name: k_DriveTrainExponent
178 //
179 // d: This is the delay value of an infinite impulse response (IIR)
180 // digital low pass filter (DLPF). Its intent is to make the motor
181 // output less "jerky" (i.e. more smooth) over time, at the
182 // expense of introducing some delay from the joystick input
183 // to the motor output.
184 //
185 // The value d must be constrained to 0.00 <= d <= 1.00.
186 //
187 // As the value of d decreases, the more smoothing (but more
188 // delay) is introduced. As d increases, less delay and
189 // less smoothing is used.
190 //
191 // NOTE: Motor controllers for the drive train should be
192 // put in "BRAKE" mode, and not in "COAST" mode for
193 // this to work properly.
194 //
195 // For more information about IIR DLP filters, and how they
196 // belong to a branch of engineering mathematics called
197 // Digital Signal Processing (DSP), see the following:
198 //
199 // https://en.wikipedia.org/wiki/Digital_signal_processing
200 // https://en.wikipedia.org/wiki/Infinite_impulse_response
201 // https://www.dsprelated.com/freebooks/filters/Introduction.html
202 // https://www.embedded.com/design/configurable-systems/4025591/Digital-filtering-without-the-pain
203 //
204 // Constant name: k_DriveTrain_IIR_DLPF_d
205 //
206 // See the SW_Design repository, and look at NewThumbsticksOutputs.xlsx
207 // and IIR_DLPF.xlsx for some graphs that illustrate these concepts.
208
