210  // ******** Joystick/Motor Curves ********
211 
212 // The joystick/motor curve parameters should make it easier to control
213 // motors with a joystick and/or thumbstick and/or other variable
214 // control (such as the triggers on some controllers).
215 //
216 // Here is what each constant/parameter means/does:
217 //
218 // Null Zone: Sometimes called deadzone. Any absolute value output from the
219 // joystick/thumbstick/trigger/etc. that is less than the
220 // null zone value will translate into zero output onto a motor.
221 // This keeps the motor from "jittering", especially if the
222 // output of the controller is noisy (or a nervous driver ;-).
223 //
224 // Typical values for a null zone are from 0.0 to 0.2.
225 // The null zone cannot exceed 1.0 (nothing will work!).
226 //
227 // Constant name: k_DriveTrainAbsNullZone
228 //
229 // Max Speed: This is the maximum speed that the motor will output
230 // (turbo off) (either forward or reverse) in non-turbo mode.
231 // Any absolute value output from the
232 // joystick/thumbstick/trigger/etc. that is greater than the
233 // maximum speed will be limited to the maximum speed
234 // (either forward or reverse) defined by this constant
235 // parameter.
236 //
237 // Set the maximum speed to the percentage of full speed desired.
238 // For example, to limit the motor to half speed, set max speed
239 // to 50% (0.50). Similar to the null zone, don't set the
240 // max speed to 0.0, or again nothing will work!
241 //
242 // Constant name: k_DriveTrainAbsValMaxSpeedNoTurbo
243 //
244 // Max Speed: This is the maximum speed that the motor will output
245 // (turbo on) (either forward or reverse) in turbo mode.
246 // Any absolute value output from the
247 // joystick/thumbstick/trigger/etc. will be multiplied
248 // by the ratio of max_speed_turbo_on to
249 // max_speed_turbo_off. By using this ratio, the maximum
250 // speed will be limited to the maximum turbo speed, but
251 // providing boosted speed at all other speeds as well.
252 //
253 // Constant name: k_DriveTrainAbsValMaxSpeedTurboOn
254 //
255 // Exponent: This is the exponent of a curve used on the joystick input
256 // to motor output mapping. Its intent is to make the motor more
257 // responsive at lower speeds, at the loss of fine control
258 // at higher speeds. An exponent of 1.0 makes the output
259 // linear (like a straight line). Higher exponents provide
260 // for better control at lower speeds, but sacrifice control
261 // at higher speeds.
262 //
263 // You'll typically want the exponent to be between 2.0-4.0.
264 // Anything past 5.0 will result in an uncontrollable robot!
265 // Do not use an exponent less than 1.0. The math breaks
266 // down, and we may open a rift to another universe.
267 //
268 // Constant name: k_DriveTrainExponent
269 //
270 // d: This is the delay value of an infinite impulse response (IIR)
271 // digital low pass filter (DLPF). Its intent is to make the motor
272 // output less "jerky" (i.e. more smooth) over time, at the
273 // expense of introducing some delay from the joystick input
274 // to the motor output.
275 //
276 // The value d must be constrained to 0.00 <= d <= 1.00.
277 //
278 // As the value of d decreases, the more smoothing (but more
279 // delay) is introduced. As d increases, less delay and
280 // less smoothing is used.
281 //
282 // NOTE: Motor controllers for the drive train should be
283 // put in "BRAKE" mode, and not in "COAST" mode for
284 // this to work properly.
285 //
286 // For more information about IIR DLP filters, and how they
287 // belong to a branch of engineering mathematics called
288 // Digital Signal Processing (DSP), see the following:
289 //
290 // https://en.wikipedia.org/wiki/Digital_signal_processing
291 // https://en.wikipedia.org/wiki/Infinite_impulse_response
292 // https://www.dsprelated.com/freebooks/filters/Introduction.html
293 // https://www.embedded.com/design/configurable-systems/4025591/Digital-filtering-without-the-pain
294 //
295 // Constant name: k_DriveTrain_IIR_DLPF_d
296 //
297 // See the SW_Design repository, and look at NewThumbsticksOutputs.xlsx
298 // and IIR_DLPF.xlsx for some graphs that illustrate these concepts.
299
